Creating Objects

Objects store data such as properties and methods.

  • Properties: Info about the object we can retrieve and use
  • Methods: functions that belong to the object

Long way to create an Object

Use {} to create an Object.

When a JavaScript variable is declared with the keyword "new", the variable is created as an object. However this isn't necessary since you can assign properties while creating it.

This creates and Object named 'course'.

var course = new Object();

You can then run commands to assign it properties using dot or bracket notation.

course.title = "JavaScript Essential Training";
course.instructor = "Name of teacher";
course.level = 1;
course["Where it is"] = "At the school";

Short way to create an Object

The short hand way is to assign the properties while creating the Object.

var course = {
  title: "JavaScript Essential Training",
  instructor: "Name of teacher",
  level: 1,
  views: 0,
}

Retrieving data from Objects

Retrieving the full Object

You can retrieve the full Object just like a variable.

console.log(course);

Retrieving properties using dot notation

You can use dot notation to retrieve a specific property.

The syntax is: objectName.propertyName

Dot notation works well for properly named properties.

console.log(course.title);

Retrieving properties using bracket notation

You can use bracket notation to retrieve a specific property.

The syntax is: objectName["propertyName"]

Bracket notation works well for when we might have to convert the property name into a string.

console.log(person["lastName"];);

Full example how ot create and retrieve Object data

This creates an Object named myBook with data.

The data is retrieved using a string template and dot notation.

The value of 'title' is then changed manually. When checked, the new value displays.

This outputs:

  • 1984 by George Orwell
  • Animal Farm by George Orwell
let myBook = {
  title: '1984',
  author: 'George Orwell',
  pageCount: 326
}

console.log(`${myBook.title} by ${myBook.author}`)

myBook.title = 'Animal Farm'

console.log(`${myBook.title} by ${myBook.author}`)

Returning 2 Objects using a function

Instead of writing several console.log statements, you can create a function to gather the data.

This returns:

  • 1984 by George Orwell
  • A Peoples History by Howard Zinn
let myBook = {
  title: '1984',
  author: 'George Orwell',
  pageCount: 326
}

let otherBook = {
  title: 'A Peoples History',
  author: 'Howard Zinn',
  pageCount: 723
}

let getSummary = function (book) {
  console.log(`${book.title} by ${book.author}`)
}

getSummary(myBook)
getSummary(otherBook)

Create and return 2 new properties using a function

This creates two new propties within the function.

  • summary
  • pageCountSummary

This returns:

  • 1984 by George Orwell
  • 1984 is 326 pages long
  • A Peoples History is 723 pages long
  • A Peoples History by Howard Zinn
let myBook = {
  title: '1984',
  author: 'George Orwell',
  pageCount: 326
}

let otherBook = {
  title: 'A Peoples History',
  author: 'Howard Zinn',
  pageCount: 723
}

let getSummary = function (book) {
    return {
      //return different summaries by creating different properties
      //set the new summary property to a template string
      summary: `${book.title} by ${book.author}`,
      //create new pageCountSummary property
      pageCountSummary: `${book.title} is ${book.pageCount} pages long`
    }
}

//bookSummary is an Object because it returns Object data from the function
let bookSummary = getSummary(myBook)
let otherBookSummary = getSummary(otherBook)

console.log(bookSummary.summary)
console.log(bookSummary.pageCountSummary)

console.log(otherBookSummary.summary)
console.log(otherBookSummary.pageCountSummary)

Change a property in an Object

This changes a single property (by reference)

Sending an Object to a function's parameter creates a reference to the Object.

Any change to the reference makes a change to the actual Object.

This outputs twice:

  • { name: 'Andrew Mead', expenses: 2.5, income: 0 }
  • { name: 'Andrew Mead', expenses: 2.5, income: 0 }

This shows the new value within the function also permanenty changed the actual Object.

let myAccount = {
  name: 'Andrew Mead',
  expenses: 0,
  income: 0
}

//The account param is passing a reference to the myAccount Object in memory
let addExpense = function (account, amount) {
  //change expenses property
  account.expenses = account.expenses + amount
  //Print value of Object within this function
  console.log(account)
}

//Send the myAccount Object into the function as the first param
//Send 2.50 to the 2nd param
addExpense(myAccount, 2.50)
//Check the Object now and see the expenses property has changed
console.log(myAccount)

This does not change the Object since it breaks the reference

You can only change properties of the Object within the function.

If you assign the Object to a new Object, it DOES NOT change the original Object because the reference is then broken.

This outputs

  • {}
  • { name: 'Andrew Mead', expenses: 2.5, income: 0 }

Using the assignment operator on the param name creates a new local Object. It breaks the reference to the actual Object.

let myAccount = {
  name: 'Andrew Mead',
  expenses: 0,
  income: 0
}

let addExpense = function (account, amount) {
  //set account (the reference) to an empty Object
  account = {}
  //Print value of Object within this function
  console.log(account)
}

addExpense(myAccount, 2.50)
console.log(myAccount)

Changing the property by reference (without a function)

By assigning a variable to the Object, you create a reference to the Object.

This outputs

  • { name: 'Andrew Mead', expenses: 2.5, income: 1000 }
  • { name: 'Andrew Mead', expenses: 2.5, income: 1000 }
let myAccount = {
  name: 'Andrew Mead',
  expenses: 0,
  income: 0
}

//Create a reference (binding) to the original Object
let otherAccount = myAccount
//Change a property of this reference
otherAccount.income = 1000

let addExpense = function (account, amount) {
  account.expenses = account.expenses + amount
  console.log(account)
}

addExpense(myAccount, 2.50)
console.log(myAccount)

Deleting an Object or property

Deleting a property

You can delete a property by running 'delete' following by the property value in dot notation.

delete course.title;

Empty an Object

You can empty an object by setting it to null.

If you now run 'person.firstName' it will return null.

var person = { firstName:"John",
  lastName:"Doe", age:50,
  eyeColor:"blue"
};

var person = null;

Object conversion

Avoid String, Number, and Boolean objects. They complicate your code and slow down execution speed.

var x = new String(); // Declares x as a String object
var y = new Number(); // Declares y as a Number object
var z = new Boolean(); // Declares z as a Boolean object

Objects in JavaScript can be converted to primitives in three contexts:

  • Numeric
  • String
  • Boolean

Iterating over Object key:value pairs

You can loop through key:value pairs by using a for loop with the 'key' keyword.

var menu = {
  width:  300,
  height: 200,
  title: "Menu"
};

for(var key in menu) {
var val = menu[key];

console.log("Key: " + key + " \nValue: " + val);
}

The code above prints out the following:

Key: width
Value: 300

Key: height
Value: 200

Key: title
Value: Menu
