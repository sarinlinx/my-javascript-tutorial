Basics of Functions

Steps to add a new DOM element.

  • All functions are defined with 'function' keyword.
  • Giving the function a name is optional.
  • Function names cannot start with a number. However, they can include: $, _, a...z, 0...9
  • You can have zero or as many parameters as you want, and these will become variables that are local to and only available inside your function.
  • Function parameters are the names listed in the function definition.
  • Function arguments are the real values received by the function when it is invoked.

3 types of Functions

Steps to add a new DOM element.

  • Named - executed when called by name. Usefull when the function needs to be called multiple times.
  • Anonymous - ran after being triggered by an event. Parenthesis are right after the word function.
  • Immediately - invoked function expressions run the moment the browser encounters them. These are anonymous functions with another () at the end to trigger them. And it's all wrapped in parenthesis.

Named

This function has a name of 'multiply'.

function multiply(a,b) {
  var result = a*b;
  console.log(result);
  return result;
}

var multiplied = multipy(3,4);

Anonymous

This function does not have a name. It's triggered after assigning a variable to it, then running that variable function name.

var divided = function() {
    var result = 3/4;
    console.log("3 divided by 4 is ", result);
}

divided();

Immediately invoked

Two ways to write the part that immediately invokes it:

  • (function () { /* ... */ })();
  • (function () { /* ... */ }());
(function() {
  var result = 12 / 0.75;
  console.log("12 divided by 0.75 is ", result);
} () )

Declaring functions

2 ways to declare a function

Function declaration

Declaring a function without a name

This is only executed when called.

function plus(a,b) {
  var sum = a+b;
  return sum;
}

//call the function
plus(2,2);

Declaring a function with a name

This is only executed when called.

function myFunction(parameter1, parameter2, parameter3) {
  code to be executed
}

Definition expression (aka 'anonymous function')

Assigns functions to variable or expression. A function expression can be stored in a variable.

Functions stored in variables do not need function names. They are always invoked (called) using the variable name.

Variable assigned but not used

This is ran when it's called. For example: plus(2,2);

This prints 4 to the console.

var plus = function(a,b) {
  return console.log(a + b);
};

//call the function
plus(2,2);

Variables used as functions

After a function expression has been stored in a variable, the variable can be used as a function.

This returns: 12

<p id="demo"></p>

var x = function (a,b) {
  return a * b;
};

document.getElementById("demo").innerHTML = x(4,3);

Another way variables can be used

The function is first named.

The variable is then assigned to the function name along with arguments.

function myFunction(a, b) {
return a * b;
}

var x = myFunction(4, 3) * 2;

Immediately invoke instead of calling it later

A function is immediately invoked by adding () after the closing }.

var plus = function(a,b) {
  return console.log(a+b);
} (2,2);

Where to call (invoke) the function?

It doesn't matter. The browser loads all JS, then runs it.

But for readability, it's easier to put the function before it's called in the script.

Where to define variables?

You must define them BEFORE the function is called.

function multiply(a,b) {
  var result = a*b;
  console.log(result);
  return result;
}

var multiplied = multipy(3,4);

multiply();
