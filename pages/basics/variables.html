Variables

vars are case sensitive.

The first character of a variable must be one of the following:

  • $
  • _
  • a letter

Declaring variables

Declare a variable that's undefined (has no value).

var carName;

You can then assign a value.

carName = "Volvo";

You can also declare several variables in a singl statement.

var person = "John Doe", carName = "Volvo", price = 200;

A declaration can span multiple lines.

var person = "John Doe",
carName = "Volvo",
price = 200;

var is no longer used

var was used in the past, but no longer used due to its quirks. It still works, but modern devs do not use it any longer.

One reason is that you can re-declare a variable with the same name to a different value. let and const do not allow this.

var is function scoped, not block scoped. If var is used in a function, you cannot access it outside of that function. If var is used in an if block statement, you can still access it outside. If let us used in the if statement, it's not accessible outside of it.

When a var is created, its declaration is hoisted to the top of the scope. So you can call that variables before it's declared and it will display as undefined. This would throw an error with let.

2 new types in ES2015

const

Constant. Cannot be changed once defined.

This causes an Uncaught TypeError: Assignment to constant variable.

This happens because you CANNOT change the constant after it's declared.

The biggest reason to use const is for code readability.

const MYCONSTANT = 5;
console.log(MYCONSTANT);
MYCONSTANT = 6;

Change value of const Object

Even though you can't change a const variable, if it's an Object, you can still change it's properties.

This returns: Object { age: 28 }

const person = {
  age: 27
}

person.age = 28
console.log(person)

let

Block scope variable. Smaller scope than var.

Notice how 'var localVar' is commented out twice.

If both were not commented, the nested if statement would changed the variable 'localVar' to "I'm different" inside the if statement and outside of it.

When 'let' is used, it only changes inside the variable inside of the if statement. This is because the if statement is the variable's 'Block Scope'.

function logScope() {
  //var localVar =2;
  let localVar = 2;

  if (localVar) {  //This is TRUE
    // var localVar = "I'm different";
    //this if statement is the block scope of the variable
    let localVar = "I'm different";
    console.log("nested localVar: ", localVar);
  }

  console.log("logScope localVar: ", localVar);
}

logScope();

Variable Scope

Scope determines where in your code your variable is available.

The location of the variable and how it is declared determines the scope.

There are 3 types of scope in JavaScript.

  • Global - This is when you declare the variable outside of any function. You can then access it in any script. However, if you update the variable within any function, the global value changes. So, any other references to that variable are updated as well.
  • Local - This is when you declare the variable inside a function. It's only then available within that function.
  • Block - new scope added with ES2015 used with 'const' and 'let' variables. The scope is limited to a small block of code, even within a function.
