Default TRUE (truthy) values

The following values are always truthy:

  • '0' (a string containing a single zero)
  • 'false' (a string containing the text “false”)
  • [] (an empty array)
  • {} (an empty object)
  • function(){} (an “empty” function)

Default FALSE (falsy) values

The following values are always falsy:

  • false
  • 0 (zero)
  • '' or "" (empty string)
  • null
  • undefined
  • NaN (e.g. the result of 1/0)

AND

AND returns last TRUE value in a series of TRUE

// Left operand is true,
// AND needs to check the right operand also
// It then returns that last TRUE value

alert(1 && 0) // 0, the last evaluated value
alert(1 && 5) // 5, the last evaluated value

AND returns FALSE if the first value is FALSE.

// Left operand is false,
// AND stops immediately and returns it

alert(null && 5) // Since the first value if FALSE (null), it is the last evaluated value

Logical OR with falsy operands

When using OR with falsy operands, it looks for the TRUE value.

In this example, the left operand is TRUE, so OR stops and returns it immediately.

The result is 1.

alert(1 || 0)

In this example, the left operand is FALSE, so OR continues to the next operand which is TRUE and returns it.

The result is 1.

alert(0 || 1)

If both are false

In this example, the left operand is FALSE, so OR continues to the next operand which is also FALSE.

Since there is no TRUE value, the result is null.

alert(0 || 1)

When to use?

This is useful when you have multiple variables and need to get the first one that is true.

This returns 'Oh-la-la' since it's the last value that isn't falsy.

var b = false
var n = 0
var s = "O-la-la"

var result = b || n || s

alert(result) // "O-la-la"

NOT

NOT converts its operand into boolean, then negates it.

alert( !true ) // returns false
alert( !0 ) // returns true

Double NOT

A Double NOT converts its value into boolean.

alert( !!"string" ) //	returns true
alert( !!null ) // returns false

Boolean conversion

Traditionally, it is possible to put a value into if. In this case, it gets converted into boolean.

if (x) {
  alert('true')
}

Artithmatic operators

Remainder

The modulous symbol % returns the remainder.

5 % 2; //returns 1

Exponentiation

For a natural number b, the result of a ** b is a multiplied by itself b times.

alert( 2 ** 2 );     // 4  (2 * 2)
alert( 2 ** 3 );     // 8  (2 * 2 * 2)
alert( 2 ** 4 );     // 16 (2 * 2 * 2 * 2)
alert( 4 ** (1/2) ); // 2 (power of 1/2 is the same as a square root, that's maths)
alert( 8 ** (1/3) ); // 2 (power of 1/3 is the same as a cubic root)

Increment/decrement

Increment/decrement can be applied only to a variable. An attempt to use it on a value like 5++ will give an error.

Operators ++ and -- can be placed both after and before the variable.

  • prefix form -When the operator stands before the variable: ++counter.
  • postfix form - When the operator goes after the variable: counter++.

The difference is the 'prefix form' returns the new value while the 'postfix form' returns the old value.

If you do not need the result of the increment/decrement, then it doesn't matter which form you use.

Prefix example

This returns the new value after increment.

Use this if you need to increase the value, and use the result of the operator right now.

let counter = 1;
let a = ++counter;

alert(a); // 2

Postfix example

This returns the old value prior to increment.

Use this if you need to increase the value, but use the previous result.

let counter = 1;
let a = counter++;

alert(a); // 1

Modify-in-place

Modifying-in-place shortens your notation.

let n = 2;
n += 5; // now n = 7 (same as n = n + 5)
n *= 2; // now n = 14 (same as n = n * 2)

alert( n ); // 14

Comma

This used mostly in Frameworks for more complex constructs.

It's odd and difficult to read so it should generally not be used.

Put several expressions in parenthesis and separate by a comma. Only the last expression is returned.

let a = (1 + 2, 3 + 4);

alert( a ); // 7 (the result of 3 + 4)

This results in TRUE since Anna is alphabetically after Abba.

Number comparison

This returns TRUE becuase 8 is less than or equal to 10.

console.log(8 <= 10)

The interpreter converts values to numbers. In this case, "01" becomes the numerical value of 1.

This results in TRUE.

alert( "01" < 2 );

This also results in TRUE because true converts to 1 in boolean.

alert(1 == true);

NOTE: Be careful with string numbers since it will go in alpha numeric order.

This results is TRUE since 2 is > than 14 when using lexigraphical order (dictionary sorting).

This is because lexicographical order puts everything in alpha numberic order. So ANY number that start with a 1 comes before ANY number that starts with a 2.

==

Two equal signs is a general equality operator.

This result is TRUE.

alert(0 == false);

Strict equality operator ===

Three equal signs is a strict equality operator. The values must match exactly.

This result is FALSE since values of different type are always unequal.

alert(0 === false);

In-equality operator !=

This is false since zero does equal a false boolean value. The type doesn't matter.

console.log(0 != false)

Strict in-equality operator !==

This is true since zero is not equal to false since they are different types.

console.log(0 !== false);

Special values

This is weird and something to remember to look out for.

null > 0 is false

null == 0 is false

but...

null >= 0 is true
